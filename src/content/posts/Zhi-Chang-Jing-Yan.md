# [近几年我在职场踩过的坑 | laike9m](https://laike9m.com/blog/jin-ji-nian-wo-zai-zhi-chang-cai-guo-de-keng,143/) 

1. 项目的坑
   1. 当需求不是直接来源于用户时一定要谨慎
   2. 警惕那些依赖其它组的项目
   3. 了解组里其它人在做什么
   4. 不要对项目产生感情
2. 老板的坑
   1. 不要盲目相信老板
   2. 警惕只想把你当工具人的老板
3. 组的坑
   1. 业务/技术/个人兴趣，至少要占一项
   2. 不要去 scope 太小和前景莫测的组
4. 换组心得
   1. 关注具体的工作内容和大方向
   2. Impact 是否好兑现
   3. 和工程师聊天
   4. 看准时机
